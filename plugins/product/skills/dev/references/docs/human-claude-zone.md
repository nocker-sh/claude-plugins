# Claude ゾーンと人間ゾーン

`signals/{slug}.md` `backlogs/{slug}.md` は、Claude が再生成する領域を先に、人間が自由に書く領域を後に置き、`---` で分ける。triage スキルは Claude ゾーンのみ書き換え、人間ゾーンには触らない。

## 構造

```
frontmatter（任意）

# {テーマ名}

Claude ゾーン（triage が再生成。手で書いた変更は次回上書きされる）

---

人間ゾーン（オプショナル。経営判断、優先度メモ、社内コメント、ステータス変更など）
```

## 境界の判定

frontmatter を除いた本文中の `---` が Claude ゾーンと人間ゾーンの境界。境界が無いファイルは全体が Claude ゾーン。

境界判定を壊さないため、Claude ゾーンの本文中に水平線 `---` を書かない（境界専用）。

## ルール

- Claude（triage スキル）は `---` より上のみ書き換える。`---` から下は読み取りのみで触らない
- 人間ゾーンはオプショナル。新規作成時に Claude は `---` を置かない。人間がメモを書きたくなったときに末尾へ `---` を置いて書き足す
- `最終更新` 行は Claude ゾーンの先頭（テーマ名の直下）に置き、再生成時に今日の日付へ更新する
- 再生成前に `---` の有無と位置を確認し、人間ゾーンを誤って上書きしない

## ファイル別の中身

各ファイルの本文構造とフィールドは [signals.md](signals.md) と [backlogs.md](backlogs.md) を参照。
